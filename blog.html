BLOG POST #2:

(I've had this written for a couple days at this point, time has hated me this week)
I took a look into it during week 1 as well, but speculation is the phase where we decide which features to work with and about how much time it’d take to implement. The way I see it, a team can use this phase to mock up parts of the features they decide to work with.
Upon discussing with the team, three of our server-side features would be variable destructuring, the inclusion of es-lint, and the promisify method using the node utility class. Es-lint will help keep us in line as far as coding practices (though some of mine will still probably show through, because I’m a bit rebellious like that), the variable destructuring would allow us to get references to properties of objects instead of needing the entire object, and I still need to look into the promisify method that Matt and Cody spoke of.
We don’t particularly have an original source for our project, our original projects didn’t quite match up to what was required without doing a complete recreation of each project, which would’ve cost us a lot more time than what we have. Because of that, a custom built project was created so we could continue onward.
